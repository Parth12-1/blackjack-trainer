# M3 Features Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy the trainer live, then add live expected-payout, a manual calculator tab, and a count-decision teaching module, plus a medium UI polish pass.

**Architecture:** All features are thin UI wrappers over existing pure engines (`montecarlo`, `strategy`, `deviations`, `countingUtils`). No new game math. New UI mounts via the existing top-nav (`App.tsx`), the CountingTrainer module switcher, and the PlayTable footer toggles.

**Tech Stack:** Vite, React 18, TypeScript (strict), Tailwind v3, Zustand, Framer Motion, Vitest.

## Global Constraints

- TypeScript strict — no `any`, no non-null `!` on engine outputs without a guard.
- No backend, no login, no real money (SPEC.md).
- Bet ramp: TC ≤ +1 → 1u, +2 → 2u, +3 → 4u, +4 → 8u, +5 → 12u (cap). Use `countingUtils.calcBetUnits`, do not re-hardcode.
- Deviation thresholds come only from `src/engine/deviations.ts` — never re-hardcode.
- `npm run build` must exit 0 and `npx vitest run` must stay green after every task.
